Refactor: 各モジュールの新旧仕様の統合とクリティカルなバグ修正

自動運転シミュレーションのテストマネージャーおよびチェッカーにおいて、新仕様（Configを用いた動的・汎用的な設計）と旧仕様（堅牢なエラーハンドリングや分かりやすいログ出力）の「いいとこ取り」を行う統合を行いました。また、長時間の実行時にシステムがクラッシュ・停止する複数の潜在的なバグを修正しました。

【主な変更点】
1. awchecker.py
- 外部ツール(aw_checkerpy.py)の呼び出しを1ファイルにつき1回に最適化し、処理速度を向上（旧仕様の復活）
- JSONファイルの書き込み破損を安全に検知・スキップするロジックを復元
- ターミナル上の見やすい統計UI（Safe/Unsafe/Errorの累計表示）を復活
- 検証式と設定の外部化（新仕様）との統合を完了



3. strategist.py
- estimatorの train() メソッドが予期せぬ戻り値やエラーを出した際に、永遠にSobol探索（ランダム探索）から抜け出せなくなる無限ループの罠
- estimator側の load_results() を経由してデータを読み込むよう修正し、CSV読み込み時の安全性を確保

4. estimator.py
- ヘッダーの有無を自動判別しConfigと同期させる強力なCSV読み込みロジック（新仕様）を適用
