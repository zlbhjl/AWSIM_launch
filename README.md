Refactor: 各モジュールの新旧仕様の統合とクリティカルなバグ修正

自動運転シミュレーションのテストマネージャーおよびチェッカーにおいて、新仕様（Configを用いた動的・汎用的な設計）と旧仕様（堅牢なエラーハンドリングや分かりやすいログ出力）の「いいとこ取り」を行う統合を行いました。また、長時間の実行時にシステムがクラッシュ・停止する複数の潜在的なバグを修正しました。

【主な変更点】
1. awchecker.py
- 外部ツール(aw_checkerpy.py)の呼び出しを1ファイルにつき1回に最適化し、処理速度を向上（旧仕様の復活）
- JSONファイルの書き込み破損を安全に検知・スキップするロジックを復元
- ターミナル上の見やすい統計UI（Safe/Unsafe/Errorの累計表示）を復活
- 検証式と設定の外部化（新仕様）との統合を完了

2. 戦略AI (`strategist.py`) のロジック刷新：境界安定性判定とマージン検証
- **課題**: 従来の「全体のグレーゾーン比率」に頼った探索終了・移行判定では、探索空間の隅にある無関係な箇所の不確実性を解消しようとし、探索が永遠に終わらない問題があった。
- **解決策**: 「境界線の安定」と「マージン領域の安全証明」という、より実践的で明確な目標に基づいた2段階のロジックを導入。
    - **(1) 境界安定性によるフェーズ移行 (STEP 2 → 3)**
        - **判定方法**: 探索空間内に固定の「観測点」を設置。AIの予測モデルが更新されるたびに、安全/危険の判定がどれだけ変化したか（反転率）を定点観測する。
        - **移行ルール**: 境界線付近の微小な揺らぎ（例: 予測値0.49 ↔ 0.51）を無視する「不感帯」を設け、それを超える大きな変化が一定期間なくなれば「境界は安定した」とみなし、次の検証フェーズへ移行する。これにより、不要な探索を打ち切り、本質的な検証に素早く移行できる。
    - **(2) マージン領域の不確実性に基づく探索終了 (STEP 3)**
        - **探索戦略**: 安定した境界線のすぐ内側にある「安全マージン領域（例: 予測値 0.3〜0.48）」に探索を集中させる。
        - **終了条件**: このマージン領域内において、AIが「結果を確信できない（不確実性が高い）」と判断する地点を徹底的に潰していく。領域内の全ての地点で不確実性（標準偏差σ）が規定値（例: 0.05）を下回った瞬間に、「マージン領域の安全性は数学的に検証完了」と判断し、システムを正常終了させる。

3. strategist.py (バグ修正)
- estimatorの `train()` メソッドが予期せぬ戻り値やエラーを出した際に、永遠にSobol探索（ランダム探索）から抜け出せなくなる無限ループの罠を解消。
- estimator側の `load_results()` を経由してデータを読み込むよう修正し、CSV読み込み時の安全性を確保。
